Inverse Kinematics with Obstacle Avoidance by Cost Minimization

Overview

These notes describe a method for creating an animation of a robot arm by (iteratively) minimizing a cost function. This is a path-planning solution that minimizes a multi-objective cost function. The arm will move from an initial position to a set of multiple goal positions. As the arm moves, it should avoid collisions with the various obstacles in the scene. The original method is from Breen's paper [1]. Figure 1 shows a pose of the robot arm as it moves from one target location to another.

The cost function and its components

The animation is achieved by using the gradient-descent algorithm to solve the following minimization problem:

(1)Φ^=argΦ⁡minC(Φ),

where the cost function C is defined by:

(2)C(Φ)=∥e(Φ)−g∥⏟goal attraction+∑i=1nFR(∥e(Φ)−oi∥)⏟obstacle-avoidance penalty+∑j=13L(ϕj)⏟Joint-range limit.

Here, g=(xg,yg,zg)T is the goal location, oi is the location of obstacle i. Function e(Φ) computes the arm's forward kinematics and returns the location of the arm's tip e=(ex,ey,ez)T, i.e., the end-effector, given the arm's joint angles, Φ=(ϕ1,ϕ2,ϕ3,ϕ4)T. Function FR is a collision-avoidance penalty field. It penalizes poses that take the end effector too close to an obstacle, i.e., beyond a pre-defined distance R. The third component of Equation 2 limits the range of each joint angle. Function L is another penalty function. Its value increases as the joint angle ϕj approaches its maximum or minimum limit. Outside these limits, L vanishes.

Next, we describe the components of Equation 2 in more detail.

The attraction term

The goal attraction term guides the animation towards the goal location. Since its equation is the Euclidean norm (or Euclidean distance), the minimal solution path when disregarding obstacles and joint range limits is a straight line from the end-effector's current location to the goal location.

(3)C1(Φ)=∥e(Φ)−g∥

The field potential function for obstacle avoidance

The field potential is a penalty term that increases its value as the end effector approaches an obstacle. It is defined as follows:

(4)FR(d)={ln⁡(R/d),0<d≤R,0,d>R.

An example of the field potential in 4 is shown in Figure 2.

Figure 2: Penalty field function for R=10. Left: One-dimensional plot of the penalty function. Penalty value increases for distances close to the obstacle. Right: 2-D representation of the field function for d=x2+y2, i.e., distance from any point (x,y) to the origin. When the distance from obstacle is larger than R, the field potential vanishes.

The range-of-motion limit function

The limit function constrains the range of motion of the joints (i.e., angles), and is given by:

(5)L(ϕ)={ln⁡(δ/(ϕ−ϕmin)),ϕmin<ϕ≤ϕmin+δ0,ϕmin+δ<ϕ<ϕmax−δln⁡(δ/(ϕmax−ϕ)),ϕmax−δ≤ϕ<ϕmax,

where ϕ is the joint angle, ϕmin and ϕmax are the limits of that joint, and δ is the angular distance from each of the limits after which the limit function vanishes. Figure 3 shows an example of the limit function for δ=45°, ϕmin=90°, and ϕmax=270°.

Inverse kinematics

The forward-kinematics function can be used to move the robot arm given a set of (known) joint angles as input. In computer graphics, forward kinematics also helps us to draw the robot arm. However, forward kinematics is of little help when we want to move the arm's end effector to a specific location in space and we do not know the joint angles that perform the desired motion. In this case, we need to compute the inverse kinematics. To compute the joint-angle configuration for a given goal position of the end effector, we use inverse kinematics, which is the inverse of the forward-kinematics function, i.e.:

(6)Φ=f−1(e).

However, this function does not usually have a closed-form solution. In addition, non-linearity will cause it to have multiple solutions. Instead of trying to solve the inverse kinematics directly, we can make progress by using an iterative numerical-minimization method such as the gradient descent.

Solving the inverse kinematics numerically

One numerical method used for computing inverse kinematics works is based on the gradient-descent approach, and attempts to find the best incremental changes in joint-angle configuration that take the end effector closer a goal position (and orientation). In the gradient-descent method, the best direction for incremental changes in the function variables is the inverse of the gradient. Because the forward-kinematics function is a vector function of vector variables, the "gradient" vector is replaced by the Jacobian matrix.

The path obtained by the gradient-descent solution is the path of the animation. Here, we want to determine how incremental changes of joint-angle configuration relate to incremental changes of end-effector location, i.e., we want to find out how to relate changes in any component of Φ with changes in any component of e. With this relationship at hand, we can derive an iterative algorithm analogous to the gradient-descent to calculate the arm's pose needed to reach a given location.

Relating variations of pose to variations of end-effector location

To relate small changes between values of a function and small variations of its variables, we can use derivatives. Indeed, the derivative of forward-kinematics function, de/dΦ, contains all the information we need to know about how to relate changes in any component of Φ to changes in any component of e. Because the forward-kinematics function e is a vector-valued function of the vector-valued variable Φ, its derivative is given by the Jacobian. The Jacobian for a robot arm with 4 joint angles is then:

(7)dedΦ=J(e,Φ)=[∂e∂ϕ1∂e∂ϕ2∂e∂ϕ3∂e∂ϕ4]=[∂ex∂ϕ1∂ex∂ϕ2∂ex∂ϕ3∂ex∂ϕ4∂ey∂ϕ1∂ey∂ϕ2∂ey∂ϕ3∂ey∂ϕ4∂ez∂ϕ1∂ez∂ϕ2∂ez∂ϕ3∂ez∂ϕ4].

The first column vector of this Jacobian, ∂e/∂ϕ1, describes how the end-effector position changes for small changes in join angle ϕ1. Similarly, the second column vector, ∂e/∂ϕ2, describes how the end-effector position changes for small changes in join angle ϕ2. Note that each component of the Jacobian matrix in Equation 7 is equivalent to a 1st-order derivative of a scalar function of a single variable, and can be approximated numerically using finite differences. The approximated Jacobian is then:

(8)J(e,Φ)≈[ΔeΔϕ1ΔeΔϕ2ΔeΔϕ3ΔeΔϕ4]=[ΔexΔϕ1ΔexΔϕ2ΔexΔϕ3ΔexΔϕ4ΔeyΔϕ1ΔeyΔϕ2ΔeyΔϕ3ΔeyΔϕ4ΔezΔϕ1ΔezΔϕ2ΔezΔϕ3ΔezΔϕ4].

for a small Δϕi,i=1,…,4, where the elements of the matrix in (8) are forward finite differences. We can also write the columns of the Jacobian matrix in terms of e(Φ) as follows:

(9)ΔeΔϕi=e(Φ+Δϕi)−e(Φ)Δϕi,

for small Δϕi, and i=1,…,4. Note that the displacement of Φ by Δϕi in the calculation of e(Φ+Δϕi) in Equation 9 is applied only to the corresponding joint angle ϕi which is a component of Φ, and not to all components. For example, the position of the end-effector displaced by Δϕ1 is:

(10)e(Φ+Δϕ1)=e([ϕ1ϕ2ϕ3ϕ4]+[Δϕ1000]).

The other displacements are calculated analogously.

For a numerical example of computing the Jacobian approximation in (8), assume that the current pose of the arm is Φ=(0,π/4,π/4,π/4)T, and that the lengths of the parts are L=(6,6,1,1)T, with the base-joint center located at p1=(0,0,0)T. Here, the local frames are described by their individual transformation matrices, and the world-to-frame transformations are computed as in the notes on forward kinematics. If we set all Δϕi=0.05°=(1/20)×π180 radians, the first column of the approximated Jacobian matrix will be:

ΔeΔϕ1=e(Φ+Δϕ1)−e(Φ)Δϕ1=e([ϕ1ϕ2ϕ3ϕ4]+[Δϕ1000])−e([ϕ1ϕ2ϕ3ϕ4])Δϕ1=e([0π/4π/4π/4]+[120×π180000])−e([0π/4π/4π/4])120×π180(11)=[5.94970.00529.5355]−[5.949709.5355]120×π180=[−0.00265.94970].

After computing all columns, the approximated Jacobian matrix is given by:

(12)J(e,Φ)≈[−0.00263.5329−0.7079−0.70745.94970000−5.9513−1.7068−0.7068].

We now have all the ingredients we need to derive a numerical algorithm similar to the gradient descent to compute inverse kinematics. The algorithm's derivation, which is described next, follows the same of the gradient-descent for scalar functions.

The gradient-descent for inverse kinematics

How pose change as a function of end-effector position?

Let ΔΦ be the vector representing a small change in joint-angle values, which causes the following change in the end effector e:

ΔeΔΦ≈dedΦΔe≈dedΦ⋅ΔΦ≈J(e,Φ)⋅ΔΦ(13)≈J⋅ΔΦ.

Equation 13 describes an incremental forward-kinematics computation. We need to calculate an incremental inverse kinematics. To do that, we need to solve Equation 13 for ΔΦ as follows:

Δe≈J⋅ΔΦ(14)ΔΦ≈J−1⋅Δe.

Often, the Jacobian matrix is not invertible. In fact, this is the case of the Jacobian matrix of the robot arm in these notes (the matrix is not a square matrix). Thus, instead of using the inverse of the Jacobian, we will calculate its pseudo-inverse. By using the Jacobian's pseudo-inverse denoted here by J†, we can re-write Equation 14 as:

(15)ΔΦ≈J†⋅Δe,

where:

(16)J†=VD+UT

is the Moore-Penrose pseudo inverse of the Jacobian. Here, matrices U, D, and V are obtained from the Singular Value Decomposition of J, i.e., J=UDVT.

Thus, given some desired incremental change in the end-effector position, we use (15) to compute an appropriate incremental change in joint-angle configuration.

Computing the value of Δe

Equation 14 depends on the value of Δe that will take the end-effector closer to the goal location. Calculating this value is the next step of our algorithm. Let Φ be the current pose of the robot arm and e be the current end-effector position. Also, let g be the goal position that we want the end effector to reach. When solving inverse kinematics using an iterative approach (i.e., gradient descent), we want to choose a value for Δe that will move e closer to g. A reasonable place to start is by setting:

(17)Δe=g−e

As in the derivation of the gradient descent for scalar functions, non-linearity prevents the exact solution that would satisfy Equation 17. Instead, we will take small steps, i.e.:

(18)Δe=λ(g−e),

with λ∈[0,1]. The basic Jacobian inverse-kinematics algorithm is listed in Algorithm 1. The diagram in Figure 4 shows an analogy between the Jacobian gradient-descent method in and the gradient-descent method for a scalar function of a scalar variable.

Minimizing the cost-function to create the animation

By using the procedure described in Algorithm 1, we can create a simple animation of the robot arm. However, this animation would be unconstrained as it would not minimize the cost function in Equation 2. To minimize the cost function that controls the animation, we can use a two-step approach. In the first step, we calculate the direction of minimal cost, which is to where we want to move the arm. In the second step, we move the arm in that direction by a fraction along the direction of minimal cost . The two steps require computations of inverse kinematics using Algorithm 1, and is illustrated in Figure 5.

Figure 5: Computing the direction of minimal cost. First, use inverse kinematics to compute the joint angles Φx,Φy,Φz that would take the end-effector to nearby locations e+Δx, e+Δy, and e+Δz. Then, compute costs C(Φx),C(Φy),C(Φz) at those locations. Next, create vector d=(C(Φx),C(Φy),C(Φz))T. Its reciprocal (i.e., −d) is the direction of minimal cost. The next (incremental) location of e in the animation is given by gnew=e−d. Finally, move the arm towards location gnew using inverse kinematics.
